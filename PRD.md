# 思源笔记内容推送到 Wiki 展示插件 PRD

## 1. 文档信息
- **产品名称**：思源笔记 Wiki Push 插件
- **版本**：v1.2（深化版）
- **目标用户**：使用思源笔记进行知识管理，并希望同步内容到 Wiki 对外展示或团队共享的个人/团队用户
- **核心目标**：将思源笔记中的文档（页面）稳定、安全地推送到目标 Wiki，并支持后续更新、版本追踪、推送管理（删除/暂停/恢复）与失败重试。

## 2. 背景与目标
### 2.1 背景
用户在思源笔记中沉淀内容，但内容对外发布、团队共享或构建公开知识库时，往往需要重复搬运到 Wiki 系统，存在：
1. 人工复制粘贴成本高；
2. 版本不同步导致信息不一致；
3. 多页面批量发布效率低；
4. 已推送页面缺乏统一管理，无法方便地下线、删除或暂停同步；
5. 网络与权限问题导致推送结果不可预期。

### 2.2 产品目标（v1.2）
1. 一键将思源文档推送到 Wiki 页面；
2. 支持增量更新（同一文档后续可覆盖更新）；
3. 支持批量推送并具备失败重试；
4. 支持管理“已推送页面”（删除、暂停、恢复、重推、解除映射）；
5. 推送过程可观测（日志、状态、错误提示、统计）；
6. 在 API 限制下保障安全与稳定（鉴权、限流、退避重试、幂等处理）。

### 2.3 非目标（v1.2）
1. 不实现双向实时同步（Wiki 改动回写思源）；
2. 不实现复杂权限编排（仅使用已有 API Key 权限）；
3. 不实现可视化富文本差异对比；
4. 不实现“回收站”式软删除（删除远端即不可恢复，恢复能力由 Wiki 原生版本体系承担）。

## 3. 角色与核心场景
### 3.1 用户角色
- **个人作者**：发布个人知识页面。
- **团队维护者**：批量推送、控制页面上下线。
- **空间管理员**：确保推送范围、权限与发布规范。

### 3.2 核心场景
1. **首次发布**：从思源创建新页面到 Wiki。
2. **持续更新**：同文档多次推送，覆盖更新目标页面。
3. **批量维护**：选中文档批量推送并查看失败清单。
4. **运营下线**：删除远端页面，或者仅解除映射。
5. **暂停维护**：页面加入“暂停推送”，后续批量任务自动跳过。

## 4. 需求范围

## 4.1 功能需求
### FR-1 鉴权与连接配置
- 支持配置 Wiki Base URL（如 `https://your-wiki.com/api/v1`）。
- 支持配置 API Key（Bearer Token）。
- 支持“测试连接”：调用 `/api/v1/auth/me` 或 `/api/v1/spaces` 验证可用性。
- 支持错误提示：401（Token 失效）、403（权限不足）、网络错误（超时/DNS）。

### FR-2 文档选择与映射
- 支持单篇推送、批量推送。
- 每篇文档建立“思源文档ID ↔ Wiki页面slug”映射并持久化。
- 若映射不存在则创建页面；映射存在且状态 `active` 则更新页面。
- 若映射状态为 `paused`，默认跳过并提示原因。

### FR-3 页面创建与更新
- 创建：`POST /api/v1/pages`
- 更新：`PUT /api/v1/pages/:slug`
- 字段映射：
  - `title` ← 思源文档标题；
  - `content` ← Markdown 内容；
  - `tags` ← 文档标签（可选）；
  - `visibility` ← 插件配置默认值；
  - `space_id` ← 插件配置值（可选）。
- 更新策略：默认“覆盖更新”；提供“仅正文更新（保留远端标题）”开关（默认关闭）。

### FR-4 推送任务与状态管理
- 提供任务队列，支持并发上限配置（默认 1，建议 1~3）。
- 任务状态：`queued`、`running`、`success`、`failed`、`paused`、`skipped`。
- 失败任务支持重试（单条重试、失败项批量重试）。
- 记录请求与结果日志（时间、操作、slug、HTTP 状态、错误码、错误信息）。

### FR-5 已推送页面管理（重点）
- 提供“已推送页面管理”视图：文档名、slug、状态、最近推送时间、最后错误。
- 单条操作：
  1. **删除远端页面**（`DELETE /api/v1/pages/:slug`）；
  2. **仅解除映射**（不删远端）；
  3. **暂停推送**（`active -> paused`）；
  4. **恢复推送**（`paused -> active`）；
  5. **立即重推**（强制触发一次更新流程）。
- 删除需二次确认，区分“删除远端”和“仅解除映射”。

### FR-6 批量管理能力（重点）
- 批量支持：暂停、恢复、删除远端、解除映射、重推。
- 批量结果必须返回：成功数/失败数/跳过数。
- 删除失败项可“仅重试失败项”，避免重复操作成功项。

### FR-7 错误处理与恢复
- 统一错误分层：网络层、鉴权层、业务层、服务端层。
- 典型错误提示：400 / 401 / 403 / 404 / 409 / 429 / 500。
- 429 / 5xx 自动指数退避重试（建议 1s/2s/4s，最多 3 次）。
- 404（页面不存在）时可一键“重建映射并新建页面”。

### FR-8 内容转换与发布策略
- 支持思源到 Markdown 的基础语法转换：标题、列表、表格、代码块、引用。
- 图片策略（v1.2）：默认沿用原链接；后续可升级为附件上传并替换。
- 对不可兼容块给出告警，不阻断主流程（可配置为阻断）。

## 4.2 非功能需求
1. **安全性**：
   - API Key 加密存储，日志脱敏；
   - 删除远端页面必须二次确认；
   - 支持“只读演练模式”（dry-run，不发写请求，仅检查可推送性）。
2. **性能**：
   - 单页面推送目标 < 3 秒（不含网络抖动）；
   - 批量 100 页稳定执行，429 情况下可自动退避。
3. **可靠性**：
   - 失败重试 + 崩溃恢复（重启后可恢复未完成队列）；
   - 幂等保障：同一任务在重试中避免重复创建。
4. **可观测性**：
   - 提供任务总览、错误聚合、最近推送统计；
   - 支持导出最近 N 条日志（JSON/CSV）。

## 5. API 对接方案（基于 api-docs.md）

## 5.1 鉴权
- Header：`Authorization: Bearer <api_key>`
- `Content-Type: application/json`

## 5.2 关键接口清单
1. 连接校验：`GET /api/v1/auth/me`、`GET /api/v1/spaces`
2. 创建页面：`POST /api/v1/pages`
3. 更新页面：`PUT /api/v1/pages/:slug`
4. 删除页面：`DELETE /api/v1/pages/:slug`
5. 页面查询：`GET /api/v1/pages/:slug`、`GET /api/v1/pages`
6. 附件上传（后续）：`POST /api/v1/attachments/upload`
7. 版本能力（后续）：`GET /api/v1/pages/:slug/versions`

## 5.3 接口调用约束
- 超时建议：连接超时 5s，请求超时 20s。
- 重试范围：仅对幂等场景或服务端可恢复错误自动重试。
- 限流处理：命中 429 后进入退避队列，暂停新建并发任务窗口。

## 5.4 数据流（单页面推送）
1. 用户触发推送。
2. 读取文档并转换 Markdown。
3. 查询映射与状态：
   - 无映射：创建页面并写入 slug；
   - 有映射且 active：更新页面；
   - paused：标记 skipped。
4. 记录日志并更新 UI。

## 5.5 已推送页面删除流
1. 管理页点击“删除远端”。
2. 弹窗确认删除范围。
3. 调用 `DELETE /api/v1/pages/:slug`。
4. 成功后将映射状态设为 `deleted`，保留审计日志。

## 5.6 暂停/恢复状态流
- 暂停：`active -> paused`，新任务不执行推送。
- 恢复：`paused -> active`，后续任务恢复执行。
- 状态切换均写入 `push_logs`，`actionType=pause/resume`。

## 6. 状态机定义（新增）

### 6.1 映射状态机（doc_mapping.syncStatus）
- `active`：可正常推送。
- `paused`：被人为暂停，批量任务跳过。
- `deleted`：远端已删除，不可直接更新。
- `unlinked`：仅解除映射，远端可能仍存在。

### 6.2 合法状态迁移
- `active -> paused`（手动暂停）
- `paused -> active`（手动恢复）
- `active/paused -> deleted`（删除远端成功）
- `active/paused -> unlinked`（仅解除映射）
- `deleted/unlinked -> active`（重新推送并生成新映射）

- 验收脚本

## 11. 验收标准（Acceptance Criteria）
1. API Key 正确时连接测试成功。
2. 首次推送创建页面成功，字段映射正确。
3. 二次推送可更新同一页面。
4. 暂停状态页面在批量推送中被跳过并有记录。
5. 恢复后页面可继续推送。
6. 删除远端成功后本地状态变更为 `deleted`。
7. 解除映射后不影响远端页面存在性。
8. 批量操作可输出成功/失败/跳过统计并支持失败重试。
9. 日志中不出现明文 API Key。

## 12. 风险与应对
1. **误删风险**：二次确认 + 危险操作样式 + 支持“仅解除映射”。
2. **限流风险**：低并发默认值 + 指数退避 + 失败项重试。
3. **数据漂移**：映射校验（404 自动修复向导）。
4. **权限不足**：设置页提示 scope 要求（write）。
5. **格式兼容**：对不兼容块给出清单，逐步适配。

## 13. 测试计划（新增）
- **功能测试**：单页推送、批量推送、暂停/恢复、删除远端、解除映射。
- **异常测试**：401/403/404/409/429/500、网络超时、中断恢复。
- **性能测试**：100 页批量、并发 1/2/3 对比。
- **回归测试**：升级配置兼容（v1.1 -> v1.2）。

## 14. 后续演进（v2+）
1. 自动同步（事件驱动/定时任务）。
2. 图片附件自动上传并替换链接。
3. 版本对比与回滚辅助。
4. Webhook 集成外部通知与索引刷新。
5. 管理员审批流与延时删除队列。
